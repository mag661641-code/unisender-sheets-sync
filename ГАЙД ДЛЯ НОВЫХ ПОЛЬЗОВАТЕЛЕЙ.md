# Гайд: как запустить скрипт переноса статистики рассылок

Этот гайд написан для тех, кто никогда не работал с кодом. Читайте по шагам, ничего не пропускайте.

---

## Что делает этот скрипт

Скрипт сам заходит в Unisender, находит нужную рассылку и переносит статистику (сколько отправлено, доставлено, прочитано и т.д.) в вашу Google-таблицу. Вам ничего не нужно копировать вручную.

---

## Шаг 1. Установить Python

Python — это язык программирования, на котором написан скрипт. Без него ничего не запустится.

1. Откройте браузер и перейдите на сайт: **https://www.python.org/downloads/**
2. Нажмите большую жёлтую кнопку **Download Python** (версия 3.11 или новее)
3. Запустите скачанный файл
4. **ВАЖНО:** на первом экране установки поставьте галочку **"Add Python to PATH"** — без этого ничего не заработает
5. Нажмите **Install Now** и дождитесь окончания установки

Проверить что Python установился:
- Нажмите Win + R, введите `cmd`, нажмите Enter
- В чёрном окне введите: `python --version`
- Должно появиться что-то вроде `Python 3.11.0` — всё хорошо

---

## Шаг 2. Установить нужные программы для скрипта

1. Нажмите Win + R, введите `cmd`, нажмите Enter
2. Скопируйте эту команду и вставьте в чёрное окно (правой кнопкой → Вставить), нажмите Enter:

```
pip install streamlit selenium gspread google-auth
```

3. Начнётся установка — подождите, это может занять 2-3 минуты. Когда снова появится мигающий курсор — всё готово.

---

## Шаг 3. Скачать файлы скрипта

1. Перейдите на страницу репозитория на GitHub
2. Нажмите зелёную кнопку **Code**
3. Выберите **Download ZIP**
4. Распакуйте архив в удобную папку (например, `Рабочий стол → скрипт`)

В папке должны быть файлы:
- `unisender_app.py`
- `parser_core.py`
- `unisender_parser.py`
- `packages.txt`
- папка `.streamlit`

## Шаг 4. Получить доступ к Google Таблице

Чтобы скрипт мог писать в вашу таблицу, нужно дать ему разрешение. Для этого используется специальный файл `credentials.json` — попросите его у человека, который настраивал скрипт.

После того как получили файл:
1. Положите `credentials.json` прямо в папку со скриптом (рядом с `unisender_app.py`)

Затем нужно открыть вашу Google Таблицу и дать доступ:
1. Откройте `credentials.json` через Блокнот
2. Найдите строку `"client_email"` — скопируйте адрес почты в кавычках (выглядит как `что-то@что-то.iam.gserviceaccount.com`)
3. Откройте вашу Google Таблицу
4. Нажмите кнопку **Настройки доступа** (или **Поделиться**) в правом верхнем углу
5. Вставьте скопированный email в поле "Добавить пользователей"
6. Выберите роль **Редактор**
7. Нажмите **Отправить**

## Шаг 5. Создать файл с вашим аккаунтом

В папке со скриптом создайте файл `accounts.json`:

1. Откройте папку со скриптом
2. Кликните правой кнопкой на пустом месте → **Создать** → **Текстовый документ**
3. Назовите его `accounts` (без расширения .txt) — если появится предупреждение про расширение, нажмите Да
4. Откройте файл через Блокнот и вставьте этот текст, заменив данные на свои:

```json
[
  {
    "name": "Мой аккаунт",
    "unisender_email": "ВАШ_EMAIL_В_UNISENDER",
    "unisender_password": "ВАШ_ПАРОЛЬ_В_UNISENDER",
    "sheet_name": "ТОЧНОЕ НАЗВАНИЕ GOOGLE ТАБЛИЦЫ",
    "worksheet": "ТОЧНОЕ НАЗВАНИЕ ЛИСТА В ТАБЛИЦЕ",
    "cols": {
      "sent": "G",
      "delivered": "H",
      "delivered_pct": "I",
      "opened": "J",
      "opened_pct": "K",
      "ctr_unique": "L",
      "ctr_pct": "M",
      "clicks_total": "N",
      "clicks_pct": "O",
      "unsub": "P",
      "unsub_pct": "Q",
      "spam": "R",
      "spam_pct": "S",
      "undelivered": "T",
      "desktop": "U",
      "tablet": "V",
      "mobile": "W",
      "weekday": "X",
      "time": "Y"
    }
  }
]
```

5. Сохраните файл (Ctrl + S)
6. Переименуйте файл — он должен называться `accounts.json` (не `accounts.json.txt`)

> **Как найти точное название таблицы и листа:**
> Откройте Google Таблицу. Название таблицы — это то, что написано в самом верху браузера и в Google Диске. Название листа — это вкладка внизу таблицы (например, "Реестр Email-рассылок СМУ").

## Шаг 6. Запустить скрипт

1. Откройте папку со скриптом
2. В адресной строке Проводника (где написан путь к папке) кликните один раз, введите `cmd` и нажмите Enter — откроется чёрное окно уже в нужной папке
3. Введите команду и нажмите Enter:

```
python -m streamlit run unisender_app.py
```

4. Подождите 5-10 секунд — должен автоматически открыться браузер со страницей приложения
5. Если браузер не открылся сам — скопируйте ссылку из чёрного окна (там будет написано `Local URL: http://localhost:8501`) и вставьте в браузер вручную

> **Важно:** чёрное окно терминала должно оставаться открытым пока вы работаете со скриптом. Как только закроете его — приложение перестанет работать.

## Шаг 7. Работа с приложением

После открытия в браузере вы увидите интерфейс:

**В левой панели:**
- **"Как дать доступ к таблице"** — тут показан email который нужно добавить в таблицу (см. Шаг 4)
- **"Добавить аккаунт"** — если нужно добавить новый аккаунт Unisender прямо из интерфейса (без редактирования файла)
- **"Удалить аккаунт"** — удалить ненужный аккаунт

**В главной части:**
- Выпадающий список — выберите нужный аккаунт
- Кнопка **Запустить** — запускает перенос данных

**Как запустить перенос:**
1. Выберите аккаунт из списка
2. Нажмите **Запустить**
3. Откроется окно браузера Chrome — скрипт сам зайдёт в Unisender
4. В приложении появится лог (журнал действий) — вы будете видеть что происходит
5. Дождитесь сообщения **"Готово"**

## Возможные проблемы

**"streamlit не является командой"**
→ Попробуйте: `python -m streamlit run unisender_app.py`

**Браузер открылся но сразу закрылся**
→ Посмотрите на текст в чёрном окне — там будет описание ошибки. Сфотографируйте и отправьте тому, кто настраивал скрипт.

**"credentials.json не найден"**
→ Убедитесь что файл `credentials.json` лежит прямо в папке со скриптом (рядом с `unisender_app.py`)

**Скрипт не находит рассылку в Unisender**
→ Проверьте что тема письма в таблице написана точно так же как в Unisender (с учётом пробелов и знаков препинания)

**В таблице появились неправильные данные**
→ Не закрывайте приложение, сообщите об этом тому, кто настраивал скрипт
